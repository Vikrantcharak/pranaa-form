Before you use this form

This version is built for field interviews, not casual self-entry. It uses proper single-select and multi-select inputs, validates required fields, captures consent, supports conditional follow-up prompts, and never claims a save succeeded unless the server confirms it.

Drafts are autosaved in the browser. If the device is offline or the server fails, the interview is queued locally so you do not lose data. You can also export a JSON backup for manual upload.

Do not collect full names unless they are genuinely required. For health-related interviews, initials or an interview ID are safer.

Expected server response The submit endpoint should return JSON like {"ok": true, "submissionId": "..."}. If it does not, the form treats the submission as failed and keeps a local backup.
